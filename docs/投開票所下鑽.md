# 投開票所層級下鑽 — 規劃與可行性（階段二）

> 狀態：**尚未實作**。主圖（村里層級）已完成；本文件說明把「重點選區」
> 下鑽到投開票所所需的資料路徑、限制與建議做法，供階段二施工。

## 為什麼村里是主、投開票所是輔

- **村里**：全台約 7,700 里，**有現成多邊形邊界**（內政部村里界圖），
  三屆涵蓋完整、可直接 choropleth。本專案主圖即此層。
- **投開票所**：每屆全台約 1.7 萬所。**沒有官方多邊形邊界** —— 投開票所是
  「投票地點」不是「行政區」，一個里常含多個投開票所，但中選會不公布其轄區界線。
  因此投開票所只能以**點位**（依投票所地址 geocode）或**併入所屬里**呈現，
  無法做成乾淨的面量圖。故定位為「重點選區下鑽」，非全台主圖。

## 資料來源（投開票所得票）

kiang 鏡像的 `data/council/` 僅整理到**村里**；`data/count/` 是政黨層級彙總，
**不含投開票所**。投開票所明細的正源是中選會官方資料庫 ZIP：

```
http://data.cec.gov.tw/選舉資料庫/votedata.zip
```

- 內含各場選舉「各投開票所候選人得票數」的 CSV（**Big5 編碼**，檔名亦 Big5）。
- 解壓需處理 cp437→big5 檔名（見 kiang repo 的 `votedata.py` 範例）。
- 涵蓋 2014 / 2018 / 2022 三屆議員選舉，粒度到投開票所（欄位含
  縣市/鄉鎮/村里/投開票所別/候選人/得票數）。

## 建議實作

1. **抓取與解碼**：下載 votedata.zip，依 `votedata.py` 方式解 Big5 檔名，
   篩出三屆「縣市議員 / 直轄市議員」的投開票所明細。
2. **選定重點選區**：不做全台（1.7 萬點過密）。由主圖挑出小黨已有票倉或
   白地潛力高的選區（例：新竹市東區、台中后里所屬選區、北市文山…），
   只對這些選區產生投開票所資料。
3. **定位投開票所**：
   - 首選 **點位**：以投票所地址（中選會另有「投票所一覽」）geocode 成經緯度，
     地圖上以圓點表示，半徑/顏色編碼小黨得票率。
   - 次選 **併入里**：把投開票所得票歸到所屬里，當作村里圖的「明細表」展開
     （點里→列出該里各投開票所小黨得票），不另畫面。
4. **整合進現有地圖**：在 `app.js` 加一個「重點選區投開票所」圖層；放大到
   該選區時才載入對應的點位 GeoJSON，避免拖慢主圖。

## 工作量與風險

- votedata.zip 體積大（數百 MB），且 Big5 解碼、欄位對應較繁瑣。
- 投票所地址 geocode 需第三方服務或內政部 TGOS，且歷屆投票所地點會變動，
  跨屆比較時投開票所對應不穩定（村里相對穩定，這也是村里為主的理由之一）。
- 建議：先確定要看的「重點選區清單」，再針對性施作，成本最低、效益最高。
